# Upstream Attribution

This bundle is a Codex-native rewrite inspired by three upstream skill packs. The goal is to preserve their practical ideas while keeping the implementation original, compact, and easy to maintain.

| Upstream source | Role in this bundle | Attribution note |
|---|---|---|
| `Wholiver/swiftui-design-skill` | Visual direction, anti-slop design review, tokens and layout language | Use as inspiration for structure and prompts, not as copied text |
| `koshkinvv/ios-agent-skills` | SwiftUI, accessibility, performance, and architecture split | Use the skill split as a model; verify upstream license before redistributing any copied material |
| `SwiftyJourney/swiftui-expert-skill` | Expert SwiftUI review heuristics and modern API guidance | Use the review checklist idea, but keep wording and examples original |

## Practical policy

- Keep all copied or adapted material clearly traceable.
- If any upstream text, code, or assets are reused later, read the upstream LICENSE file first and keep the attribution visible.
- Do not copy entire READMEs into the final skills; summarize and rewrite them into Codex-specific instructions.
