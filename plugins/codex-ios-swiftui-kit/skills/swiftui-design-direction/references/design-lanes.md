# SwiftUI Design Lanes

Use these lane presets to keep visual decisions consistent across screens.

## Calm Productivity

- Use for: task focus, low-noise operations
- Visual logic: high spacing rhythm, muted surfaces, one accent action
- Typography: larger `title3`, restrained body text
- Color: neutral background, one accent
- Spacing: consistent sections, breathing room
- Motion: gentle reveal only
- Avoid: decorative clusters and overloading metadata

## Dense Pro Utility

- Use for: dispatch, filtering, management dashboards
- Visual logic: strong hierarchy and dense rows
- Typography: compact but readable sizes, clear labels
- Color: neutral with high contrast for actions/states
- Spacing: tighter vertical rhythm, clear separators
- Motion: minimal
- Avoid: extra glass, gradients, non-essential badges

## Editorial Detail

- Use for: detail reading, review, explanation
- Visual logic: deliberate reading flow top-down
- Typography: bigger content scale and clear caption system
- Color: desaturated palette with semantic emphasis
- Spacing: generous paragraph rhythm
- Motion: slow transitions between sections
- Avoid: button crowding and side-panel visual noise

## Conversational Assistant

- Use for: messaging, request/response loops
- Visual logic: composer-first with stream-first list
- Typography: short scannable labels, stable baseline rhythm
- Color: one accent for action and status
- Spacing: anchored input region, safe touch heights
- Motion: message arrival feedback with minimal shift
- Avoid: stacked banners above primary timeline

## Control Surface

- Use for: preferences, switches, permissions
- Visual logic: sectioned clusters with clear explanatory text
- Typography: label-first clarity
- Color: neutral controls, high-contrast warnings
- Spacing: uniform row height and grouped controls
- Motion: mostly static
- Avoid: dynamic row heights that cause jumpy scroll

## Commerce / Conversion

- Use for: calls to action and onboarding steps
- Visual logic: one high-signal primary action, one secondary action
- Typography: clear hierarchy to highlight completion steps
- Color: restrained, one persuasive accent
- Spacing: visual urgency through density and grouping
- Motion: contextual confirmation
- Avoid: urgent color overload

## Data Dashboard

- Use for: status, metrics, health, and feed status
- Visual logic: high-to-low priority row ordering
- Typography: metrics first, supporting context in secondary level
- Color: semantic state colors only where needed
- Spacing: repeated card rhythm
- Motion: subtle state change micro-feedback
- Avoid: mixed units and ambiguous metric labels
